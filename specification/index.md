# Specification

This directory is the normative source of truth for CAP.

It defines the protocol contract for:

- capability disclosure
- causal semantics
- message envelopes
- verb contracts
- conformance
- extension boundaries

Explanatory onboarding and implementation guidance belong in `documentation/`.
Machine-readable schemas belong in `schema/`.

The current public reference adapter is narrower than the long-form draft in several places. This specification keeps the CAP boundary explicit rather than collapsing the protocol down to today's adapter surface.
